# TYC Partner

Internal Flutter (iOS) app for retail store employees to process golf club
trade-ins on behalf of **TradeYourClubs**. Dark, high-contrast, built for speed
at a retail counter.

- **Stack:** Flutter (stable) · Supabase (auth / Postgres + RLS / storage) ·
  `webview_flutter` · `flutter_dotenv`
- **Bundle ID:** `com.tradeyourclubs.partner`
- **Distribution:** iOS only, TestFlight

## App structure

Bottom nav, three tabs:

1. **Check Value** — full-screen WebView of `https://valueguide.pga.com/trade-in/`
   with a reload button.
2. **New Trade-In** — intake flow (placeholder for now).
3. **History** — past trade-ins / settlements (placeholder for now).

Auth is Supabase email/password. Every user belongs to one partner (store) and
only ever sees their own store's data, enforced by Row Level Security.

## First-time setup

### 1. Secrets

```bash
cp .env.example .env
# then paste the Supabase anon key into .env
```

`.env` is gitignored. It holds:

```
SUPABASE_URL=https://ovwiermvrcdtazqtyoga.supabase.co
SUPABASE_ANON_KEY=...
```

### 2. Database

In the Supabase dashboard SQL editor:

1. Paste and run [`supabase/migration.sql`](supabase/migration.sql) — creates
   tables, RLS policies, and the `item-photos` storage bucket.
2. Paste and run [`supabase/seed.sql`](supabase/seed.sql) — creates the
   "Test Golf Shop" partner, then follow its inline steps to create your clerk
   login in the Auth dashboard and link a `profiles` row.
3. Paste and run [`supabase/migration2_admin.sql`](supabase/migration2_admin.sql)
   — adds the TYC admin role + cross-partner RLS, and includes the one-liner to
   promote yourself to admin (which reveals the in-app Admin tab).
4. Paste and run [`supabase/migration3_visits.sql`](supabase/migration3_visits.sql)
   — adds `visit_id` so a customer trading in several clubs at once is saved as
   one visit (shared signature + acceptance, combined offer).
5. Paste and run [`supabase/migration4_partners.sql`](supabase/migration4_partners.sql)
   — adds partner contact fields + a "TYC HQ" partner for admin accounts.
6. **Edge Function** (needed for the Admin → Add account feature): in the
   Supabase dashboard → **Edge Functions** → **Deploy a new function**, name it
   exactly `admin-create-user`, paste
   [`supabase/functions/admin-create-user/index.ts`](supabase/functions/admin-create-user/index.ts),
   and Deploy. No secrets to set — Supabase injects the service-role key. This
   function is the only place the service-role key is used; it verifies the
   caller is a TYC admin before creating accounts.

### 3. iOS project + run (on a Mac)

The repo ships without an `ios/` folder so it's generated by your local Flutter
version — the most reliable path for a device / TestFlight build.

```bash
bash tool/setup_ios.sh   # flutter create + pub get + set bundle id / name + pod install
flutter run              # on a connected device
```

Then in Xcode (`ios/Runner.xcworkspace`): set your signing Team, confirm the
bundle id is `com.tradeyourclubs.partner`, and **Product → Archive** for
TestFlight.

## Project layout

```
lib/
  main.dart                 # loads .env, initializes Supabase, runs the app
  app.dart                  # MaterialApp + dark theme + AuthGate
  config/env.dart           # typed .env accessors (flutter_dotenv)
  theme/app_theme.dart      # design tokens + ThemeData
  services/auth_service.dart# Supabase auth wrapper
  widgets/
    auth_gate.dart          # login vs. shell based on session
    placeholder_screen.dart # shared "coming next" screen
  data/                     # hardcoded brands + club/spec/condition config
  models/                   # trade_in_draft, intake_item
  screens/
    login_screen.dart       # email/password sign-in
    home_shell.dart         # bottom nav; 3 tabs for clerks, +Admin for admins
    check_value_screen.dart # PGA value guide WebView (locked to the tool)
    new_trade_in/           # 5-step intake stepper + steps/ + confirmation
    history/                # partner history list, item detail, shared rows
    admin/                  # admin portal: pickups + team (admins only)
supabase/
  migration.sql             # schema + RLS + storage (idempotent)
  seed.sql                  # test partner + login setup instructions
```
